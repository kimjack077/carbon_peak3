| 2024 |                   … |               … |                          … |                     … |

说明：
- GDP 建议用**不变价**（例如 2015 年可比价，单位：**百亿元**）或选择你们内部口径；只要一处固定口径，模型会统一。
- emissions_mtco2：**百万吨 CO₂**。
- intensity_tco2_per_10k_cny：每万元 GDP 排放（吨 CO₂/万元），可作冗余字段用于校验。

约束：同年四项相加=100（±0.1 容差）。

# 三、场景参数（示例与边界）

- **GDP 增长率（年）**：`gdp_growth_rate`，区间建议 [2%, 7%]，也可按阶段设定（2025–2030、2030–2040…）。
- **强度下降率（年）**：`intensity_decline_rate`，区间 [2%, 8%]，亦可设置“政策加严路径”（分段变化）。
- **能源结构目标**（到某年逐步线性/非线性收敛）：
  - `renewables_target_pct_by_year`（如：2040年 55%）
  - `coal_cap_or_decline_rate`（煤比例/绝对量约束）
  - `oil/gas` 同理
- **电气化（电力需求占比）提升率**：`electrification_increase_rate` 或 目标占比与达成年份。
- **排放因子改进**（可选）：电力碳强度随可再生占比下降、气电替煤等的组配影响。

> 模型支持**多组参数**并行计算，前端以“情景卡片”展示（如：基准/政策/强化/零碳技术突破）。

# 四、算法原理

## 4.1 Kaya 恒等式（Kaya Identity）

\[
\text{CO₂}_t \;=\; P_t \times \frac{GDP_t}{P_t} \times \frac{E_t}{GDP_t} \times \frac{CO₂_t}{E_t}
\]

- \(P_t\)：人口；若未建模人口，可合并到 **GDP** 与 **能耗强度**路径。
- \(\frac{E_t}{GDP_t}\)：能源强度（能源/单位GDP）。
- \(\frac{CO₂_t}{E_t}\)：排放因子（每单位能源的排放），由**能源结构**与各能源品种的排放因子加权得到。
- 在工程实现中，可用**GDP 路径 + 综合碳强度路径**（吨CO₂/万元）替代人口与能源强度两环，以减少口径依赖：
  \[
  \text{CO₂}_t = GDP_t \times \text{Intensity}_t
  \]
- **Intensity_t** 又由电力碳强度、终端燃料结构、电气化程度、终端效率等共同决定。

## 4.2 LEAP 风格情景推演（分解法）

LEAP 的关键思想是**自上而下的活动水平（Activity） × 技术/燃料结构（Structure） × 效率/排放因子（Intensity）**逐层相乘并逐年外推。为可落地实现，我们抽象为：

\[
\text{CO₂}_t = \sum_{s \in \{\text{coal, oil, gas, power, renew}\}} A_{s,t} \times \frac{E_{s,t}}{A_{s,t}} \times EF_{s,t}
\]

- \(A_{s,t}\)：部门或用途活动量（可用 GDP*行业权重或总能耗份额近似）。
- \(E_{s,t}/A_{s,t}\)：能效项；在简化版里吸收进**份额与强度**的年度演化。
- \(EF_{s,t}\)：排放因子（例如：煤≈94.6 kgCO₂/GJ、油≈73.3、气≈56.1；电力则取**电网平均排放因子**，随可再生增长下降）。

**实现策略（工程可控）**：
- 将**能源结构份额**与**电力碳强度**作为政策杠杆逐年更新；燃料排放因子固定或按技术进步给出下降率。
- 电气化提升→把终端油/煤部分需求转移到“电力”端，同时电力端的排放随可再生占比降低而下降（双重效应）。

# 五、达峰/中和年份判定

- **达峰年**：排放序列 \(CO₂_t\) 的全局最大值第一次出现的年份，且其后5年不再超过该峰值（抗噪门槛可设1–2%的容忍区间）。
- **中和年**：\(\text{CO₂}_t \le \epsilon\)（如接近零或抵消后净零），或**电力+碳捕集**与碳汇抵消后达到净零阈值。

---

# 六、达峰目标年自动推荐（优化器思路）

**目标**：给定目标年 \(Y^*\)（默认2030），找到参数向量 \(\theta\)（强度下降率、可再生占比目标、电气化增速、煤油气份额路径、GDP增速…），使：
1) 预测达峰年 \(Y_{peak}(\theta)\) 接近 \(Y^*\)，
2) 代价最小（偏离当前政策的幅度、技术可行性、调整速度惩罚），
3) 满足约束（份额和=100%、年变化速率上限、经济增速下界、技术边界）。

**一个实用且易实现的策略**：
- **网格/拉丁超立方采样**在合理边界内生成 1e3–1e4 组 \(\theta\)；
- 逐组仿真得到 \(Y_{peak}\)、排放路径与关键KPI；
- 目标函数：
  \[
  \min_\theta \; w_1|Y_{peak}(\theta)-Y^*| + w_2 \cdot \text{Overshoot}_{post-peak} + w_3 \cdot \text{ChangeCost}(\theta)
  \]
- 取最优若干解（Pareto 前沿），前端展示“推荐场景”。

**情景引擎核心（Kaya + LEAP 式）**

```python
# services/engine.py
import numpy as np
import pandas as pd

DEFAULT_EF = {  # kgCO2/GJ（可根据口径调整）
    "coal": 94.6, "oil": 73.3, "gas": 56.1, "power": None  # power 由电网因子给出
}

class ScenarioEngine:
    def __init__(self):
        self.baseline_years = None
        self.base_macro = None
        self.base_mix = None
        self._last = None

    def load_baseline(self, df_macro: pd.DataFrame, df_mix: pd.DataFrame):
        self.base_macro = df_macro.set_index("year").sort_index()
        self.base_mix = df_mix.set_index("year").sort_index()
        self.baseline_years = list(self.base_macro.index)
        # 可加入：强度=排放/GDP 一致性校验

    def _interpolate_path(self, start, target, years):
        return np.linspace(start, target, years)

    def run(self, params: dict):
        # ===== 1) 读基期（以2024为起点） =====
        y0 = max(self.baseline_years)
        gdp0 = self.base_macro.loc[y0, "gdp_billion_2015cny"]
        e0   = self.base_macro.loc[y0, "emissions_mtco2"]
        elec_share0 = self.base_macro.loc[y0, "electricity_share_pct"]

        mix0 = self.base_mix.loc[y0, ["coal_pct","oil_pct","gas_pct","renewables_pct"]].to_dict()

        # ===== 2) 构建年度轴 =====
        end_year = params.get("end_year", 2060)
        years = list(range(y0, end_year+1))

        # ===== 3) 生成路径：GDP、强度、电气化、能源结构 =====
        g = params.get("gdp_growth_rate", 0.045)  # 4.5%
        intensity_decline = params.get("intensity_decline_rate", 0.05)  # 5%
        # 目标份额及达成年份（线性收敛）
        renew_target = params.get("renewables_target_pct", mix0["renewables_pct"])
        renew_target_year = params.get("renewables_target_year", 2040)
        electrif_target = params.get("electricity_share_target_pct", elec_share0)
        electrif_target_year = params.get("electricity_share_target_year", 2040)

        # 份额边界控制：煤/油/气 按给定下降率或自动闭式调整
        coal_decline = params.get("coal_pct_decline_per_year", 0.01)  # 百分点/年
        oil_decline  = params.get("oil_pct_decline_per_year", 0.003)
        gas_cap      = params.get("gas_pct_cap", None)  # 上限（可为空）

        # 电网排放因子路径（kgCO2/kWh 或 tCO2/MWh），此处简化用可再生占比反推
        grid_ef0 = params.get("grid_ef0_t_per_mwh", 0.55)  # 2024年基准
        grid_ef_min = params.get("grid_ef_floor_t_per_mwh", 0.05)

        # ===== 4) 逐年演化 =====
        rows = []
        gdp = gdp0
        intensity = e0 / gdp0  # tCO2 / (百亿元) ——与口径一致
        for y in years:
            if y > y0:
                gdp *= (1 + g)
                intensity *= (1 - intensity_decline)

            # 能源结构：可再生线性收敛、电气化线性收敛、煤油气按衰减并归一
            years_to_target = max(1, renew_target_year - y0)
            renew_y = mix0["renewables_pct"] + (renew_target - mix0["renewables_pct"]) * (y - y0) / years_to_target
            renew_y = max(0, min(100, renew_y))

            years_to_elec = max(1, electrif_target_year - y0)
            elec_share = elec_share0 + (electrif_target - elec_share0) * (y - y0) / years_to_elec
            elec_share = max(0, min(100, elec_share))

            coal = max(0, mix0["coal_pct"] - coal_decline * (y - y0))
            oil  = max(0, mix0["oil_pct"]  - oil_decline  * (y - y0))
            gas  = max(0, mix0["gas_pct"])
            if gas_cap is not None:
                gas = min(gas, gas_cap)

            # 归一化：化石三者与可再生合计=100
            fos = max(1e-6, coal + oil + gas)
            left_for_fos = max(0, 100 - renew_y)
            coal = coal / fos * left_for_fos
            oil  = oil  / fos * left_for_fos
            gas  = gas  / fos * left_for_fos
            renew = 100 - (coal + oil + gas)

            # 电网排放因子随可再生占比下降（简单线性近似）
            grid_ef = max(grid_ef_min, grid_ef0 * (1 - (renew - mix0["renewables_pct"])/100 * 0.9))

            # Kaya 简化：CO2 = GDP * 强度；强度由结构+电网因子修正（半经验）
            structure_factor = (coal*0.9 + oil*0.7 + gas*0.5 + renew*0.05) / 100  # 经验权重
            electrif_factor  = (1 - elec_share/100) + (elec_share/100) * (grid_ef/0.55)  # 把终端燃料替换到电力端的影响
            intensity_eff = intensity * structure_factor * electrif_factor

            emissions = gdp * intensity_eff  # 与基期口径一致（返回单位与基期相同）

            rows.append({
                "year": y, "gdp": gdp, "intensity_eff": intensity_eff,
                "emissions": emissions,
                "mix_coal": coal, "mix_oil": oil, "mix_gas": gas, "mix_renew": renew,
                "electricity_share": elec_share, "grid_ef_t_per_mwh": grid_ef
            })

        df = pd.DataFrame(rows).set_index("year")
        self._last = df
        return df.reset_index().to_dict(orient="records")

    def detect_peak(self, traj):
        df = pd.DataFrame(traj)
        y_of_max = int(df.loc[df['emissions'].idxmax(), 'year'])
        # 反弹校验（后5年不得超过峰值的101%）
        try:
            peak_val = df['emissions'].max()
            post = df[df['year'] >= y_of_max].head(6)['emissions']
            if (post > 1.01*peak_val).any():
                # 若不满足，返回None或继续向后搜索首次稳定峰
                pass
        except Exception:
            pass
        return y_of_max

    def detect_neutral(self, traj, epsilon=50):  # 单位与口径一致
        df = pd.DataFrame(traj)
        nz = df[df['emissions'] <= epsilon]
        return int(nz['year'].iloc[0]) if not nz.empty else None

    def last_result_for_viz(self):
        if self._last is None:
            return {}
        return self._last.reset_index().to_dict(orient="records")
```

```python
# services/optimize.py
import numpy as np

def recommend_scenarios(engine, target_year, bounds, nsamples=3000):
    # bounds: {param: (low, high)}
    def sample_one():
        params = {}
        for k,(lo,hi) in bounds.items():
            params[k] = np.random.uniform(lo, hi)
        return params

    cand = []
    for _ in range(nsamples):
        params = sample_one()
        traj = engine.run(params)
        y_peak = engine.detect_peak(traj)
        # 代价函数
        penalty_year = abs(y_peak - target_year)
        # 变化速率惩罚（例：强度与煤降幅越大代价越高）
        chg_cost = 0.0
        if "intensity_decline_rate" in params:
            chg_cost += max(0, params["intensity_decline_rate"] - 0.06)*100
        if "coal_pct_decline_per_year" in params:
            chg_cost += max(0, params["coal_pct_decline_per_year"] - 0.02)*50
        score = penalty_year*10 + chg_cost
        cand.append((score, params, y_peak, traj))

    cand.sort(key=lambda x: x[0])
    top = cand[:10]
    # 返回精简结果（方便前端列出推荐场景）
    return [{
        "rank": i+1,
        "score": float(s),
        "peak_year": yp,
        "params": p,
        "highlights": {
            "renewables_2040": p.get("renewables_target_pct", None),
            "intensity_decline_rate": p.get("intensity_decline_rate", None),
            "coal_decline": p.get("coal_pct_decline_per_year", None)
        }
    } for i,(s,p,yp,_) in enumerate(top)]
```

# services/validators.py
def validate_inputs(df_macro, df_mix):
    issues = []
    must_years = {2022, 2023, 2024}
    if set(df_macro['year']) != must_years or set(df_mix['year']) != must_years:
        issues.append("macro.csv 和 energy_mix.csv 必须包含 2022–2024 三年且仅三年。")
    for y, row in df_mix.set_index("year").iterrows():
        s = row["coal_pct"] + row["oil_pct"] + row["gas_pct"] + row["renewables_pct"]
        if abs(s - 100) > 0.1:
            issues.append(f"{y} 年能源份额和不为 100（= {s:.2f}）。")
    ok = (len(issues) == 0)
    return ok, issues

# 十、如何把“LEAP 思路”与“Kaya 计算”结合到代码里

1) **用 Kaya 做“核算骨架”**：CO₂ = GDP × Intensity。  
2) **用 LEAP 的“结构化杠杆”塑造 Intensity 的演化**：  
   - 终端燃料结构（煤/油/气/电/可再生）→决定排放因子加权；  
   - 电气化程度↑ → 终端燃料替换到电力端；  
   - 电网碳强度随可再生占比下降；  
   - 效率提升可作为额外的下降项（可扩展）。  
3) 代码中 `structure_factor` 与 `electrif_factor` 就是这种**可解释的合成强度**（可逐步替换为更精细的部类分项、行业负荷、技术路线与寿命更替模型）。

> 在后续版本里，可把电力侧拆成“装机-发电-上网”的技术组合（煤电/气电/风/光/核/储能/CCUS），用**容量因子与退役/新增**曲线驱动电网EF，而不是线性近似。
