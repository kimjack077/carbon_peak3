# 前端图表显示问题修复说明

## 问题描述

前端显示空白，控制台报错：
```
TypeError: Cannot read properties of undefined (reading 'map')
    at VueComponent.initCarbonPeakChart
```

## 根本原因

1. **数据缺失**：`kaya_model_fixed.py` 生成的结果文件中缺少 `power_share` 列
2. **后端错误**：`/api/chart-data` 端点尝试访问不存在的 `power_share` 列，导致返回错误数据
3. **前端脆弱**：前端代码没有对数据进行防御性检查，直接访问可能为 undefined 的属性

## 修复内容

### 1. 后端修复 (`backend/app.py`)

**文件**: `backend/app.py`  
**位置**: 第 394-413 行  
**修改内容**:

```python
# 修改前：直接访问 power_share 列
chart_data[scenario_name]['energy_mix'] = {
    'coal': (energy_total * combined_df['mix_coal']).tolist(),
    'elec': (energy_total * combined_df['power_share']).tolist(),  # ❌ 会报错
    'other': (energy_total * (combined_df['mix_oil'] + combined_df['mix_gas'] + combined_df['mix_ren'])).tolist()
}

# 修改后：检查列是否存在
if 'power_share' in combined_df.columns:
    # 如果有 power_share 列
    chart_data[scenario_name]['energy_mix'] = {
        'coal': (energy_total * combined_df['mix_coal']).tolist(),
        'elec': (energy_total * combined_df['power_share']).tolist(),
        'other': (energy_total * (combined_df['mix_oil'] + combined_df['mix_gas'] + combined_df['mix_ren'])).tolist()
    }
else:
    # 如果没有 power_share 列，使用简化的能源结构
    # 煤炭、油气、可再生能源
    chart_data[scenario_name]['energy_mix'] = {
        'coal': (energy_total * combined_df['mix_coal']).tolist(),
        'elec': (energy_total * (combined_df['mix_oil'] + combined_df['mix_gas'])).tolist(),  # 油气合并
        'other': (energy_total * combined_df['mix_ren']).tolist()
    }
```

**效果**: 后端现在可以正确处理缺少 `power_share` 列的情况，返回有效的能源结构数据。
