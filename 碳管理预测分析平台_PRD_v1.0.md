## 1. 文档说明与背景

### 1.1 编写目的

本文档为"碳管理预测分析平台"的产品需求说明书，旨在为开发团队提供完整、清晰、可执行的功能规格与技术要求，确保系统按期高质量交付。

本平台面向央企碳资产管理人员，通过历史能耗数据建立双模型预测体系，支持多情景分析与碳达峰时间识别，为企业碳中和路径规划提供科学决策支持。

### 1.2 背景与动机

我国已明确提出"2030年前碳达峰、2060年前碳中和"的双碳目标。作为高能耗央企，需要：

- 量化未来碳排放轨迹，识别碳达峰时间节点
- 模拟不同政策情景下的排放变化
- 制定科学的能源结构转型策略
- 满足监管机构对碳核查与信息披露的合规要求

本系统基于该企业 2022、2023、2024 三年历史数据进行建模，使用 LEAP 和 KAYA 两种主流算法进行情景推演。

### 1.3 核心数据

本系统**仅使用以下四项关键数据字段**（其余字段不纳入本期分析）：

| 年份 | 综合能源消费量（万吨标准煤） | 工业总产值—现价（万元） | CO₂排放量合计（万吨CO₂当量） |
|---|---|---|---|
| 2022年 | 980.15 | 8,878,582.95 | 4,818.77 |
| 2023年 | 1,098.94 | 9,961,672.21 | 5,351.33 |
| 2024年 | 1,081.95 | 10,404,978.32 | 5,273.54 |

### 1.4 术语说明

| 术语 | 定义 |
|---|---|
| 碳达峰 | 二氧化碳排放量在某年达到历史最高值，此后持续下降的转折点 |
| LEAP 模型 | Long-range Energy Alternatives Planning System，长期能源替代规划系统 |
| KAYA 模型 | Kaya Identity，碳排放恒等式分解模型（卡亚恒等式） |
| 情景 | 在设定的一组假设参数下，对未来状态的一种可能性描述 |
| 能源效率改善率 | 单位产值所消耗的能源量每年下降的百分比 |
| 可再生能源占比提升率 | 可再生能源在综合能源消费中所占比例每年提升的百分比 |
| 碳排放因子 | 消耗单位能源所产生的CO₂排放量 |
| 能源强度 | 单位工业总产值所消耗的综合能源量（万吨标准煤/万元） |
| 碳强度 | 单位工业总产值所产生的CO₂排放量（万吨CO₂/万元） |

## 2. 产品概述

### 2.1 产品定位

本平台定位为企业级碳排放预测与情景分析工具，核心价值在于：

- **双模型并行**：同时运行 LEAP 和 KAYA 两种算法，交叉验证预测结果
- **情景驱动决策**：支持多情景自定义配置，对比不同路径下的碳排放轨迹
- **可视化洞察**：直观展示碳达峰时间、能源结构演变、产值增长趋势
- **合规支撑**：输出可用于碳报告和监管申报的标准化数据

|---|---|---|

| 合规/监管对接人员 | 对接外部监管和碳核查 | 导出标准报告 |

### 2.3 产品范围

**本期（v1.0）范围：**

- 历史数据管理（2022-2024年三年数据）

- KAYA 模型情景模拟引擎

## 3. 算法规格详解

### 3.1 LEAP 模型（长期能源替代规划模型）

#### 3.1.1 理论基础

LEAP（Long-range Energy Alternatives Planning System）是由斯德哥尔摩环境研究所开发的自底向上能源-环境-经济综合模型。其核心思路是通过分析能源供需结构的演变，预测未来能耗与排放。

> **LEAP 模型核心公式**
>
> ```
> 碳排放量 = 能源消费量 × 碳排放因子 × (1 - 可再生能源占比)
>
> 其中：
>   能源消费量(t) = 基准年能耗 × (1 + 产值增长率)^Δt × (1 - 能效改善率)^Δt
>   碳排放因子    = 固定值（由基准年数据反算）
>   可再生能源占比(t) = 基准年占比 + 可再生能源占比提升率 × Δt
>   Δt = 预测年份 - 基准年（2024）
> ```

#### 3.1.2 基准参数计算（从历史数据推导）

系统使用 2022-2024 年数据拟合以下基准参数：

| 参数名称 | 计算方法 | 2024年基准值（示例） |
|---|---|---|
| 基准年能源消费量 E₀ | 直接取 2024 年数值 | 1,081.95 万吨标准煤 |
| 基准年碳排放量 C₀ | 直接取 2024 年数值 | 5,273.54 万吨CO₂当量 |
| 基准碳排放因子 f₀ | C₀ / (E₀ × (1 - R₀)) | 由历史数据反算 |
| 基准可再生能源占比 R₀ | 由用户设定初始值（默认 5%） | 5%（可配置） |
| 历史能效改善率 | (EI₂₃ - EI₂₄) / EI₂₃，EI = 能耗/产值 | 基于历史推算 |
| 历史产值增长率 | (GDP₂₄ - GDP₂₃) / GDP₂₃ | 基于历史推算 |

#### 3.1.3 LEAP 情景预测算法（逐年迭代）

系统按年度逐步迭代，每年更新以下状态变量：

1. 计算当年产值：`GDP(t) = GDP(t-1) × (1 + g)`，g 为产值增长率
2. 计算当年能源强度：`EI(t) = EI(t-1) × (1 - η)`，η 为能效改善率
3. 计算当年能源消费：`E(t) = GDP(t) × EI(t)`
4. 计算当年可再生能源占比：`R(t) = min(R(t-1) + r, 0.95)`，r 为可再生能源占比提升率
5. 计算当年碳排放：`C(t) = E(t) × f₀ × (1 - R(t))`
6. 识别碳达峰：当 `C(t) < C(t-1)` 且 `C(t) < C(t-2)`，则 `t-1` 年为碳达峰年份

> **可再生能源约束说明**
> - 可再生能源比例上限设为 **95%**（预留5%不可替代传统能源）
> - 可再生能源消耗的碳排放因子设为 **0**（零碳能源假设）
> - 非可再生能源消耗 = `E(t) × (1 - R(t))`，使用 f₀ 计算碳排放

### 3.2 KAYA 模型（卡亚恒等式模型）

#### 3.2.1 理论基础

KAYA 恒等式由日本学者茅阳一（Yoichi Kaya）于 1990 年提出，将碳排放量分解为四个驱动因子的乘积，用于分析各因子对碳排放的贡献与影响。

> **KAYA 恒等式**
>
> ```
> CO₂ = P × (GDP/P) × (E/GDP) × (CO₂/E)
>
> 即：CO₂ = 人口 × 人均GDP × 能源强度 × 能源碳强度
>
> 简化版（本系统，企业层面不考虑人口因子）：
>   CO₂(t) = GDP(t) × EI(t) × CI(t)
>
>   GDP(t) = 工业总产值（现价），万元
>   EI(t)  = 能源强度 = 能源消费量 / 产值，万吨标煤/万元
>   CI(t)  = 碳强度（含能源结构修正）= (1 - R(t)) × f₀
> ```

#### 3.2.2 KAYA 参数分解（基准年标定）

| KAYA 因子 | 符号 | 基准年（2024）计算 | 数值 |
|---|---|---|---|
| 工业总产值（现价） | GDP₀ | 直接取值 | 10,404,978.32 万元 |
| 能源强度 | EI₀ | 能耗 / 产值 = 1081.95 / 10,404,978.32 | ≈1.04×10⁻⁴ 万吨标煤/万元 |
| 基准碳排放因子 | f₀ | 碳排放 / (能耗 × (1-R₀)) | 由数据反算 |
| 碳强度 | CI₀ | EI₀ × f₀ × (1 - R₀) | ≈0.506×10⁻³ 万吨CO₂/万元 |

#### 3.2.3 KAYA 情景预测算法

1. `GDP(t) = GDP₀ × (1 + g)^(t - t₀)`，g 为产值增长率
2. `EI(t)  = EI₀ × (1 - η)^(t - t₀)`，η 为能效改善率（EI 每年下降）
3. `R(t)   = min(R₀ + r × (t - t₀), 0.95)`，r 为可再生能源占比提升率
4. `CI(t)  = f₀ × (1 - R(t))`（随可再生能源增加而降低）
5. `E(t)   = GDP(t) × EI(t)`（总能耗）
6. `C(t)   = GDP(t) × EI(t) × CI(t)`（碳排放量）
7. 碳达峰判断：寻找 `C(t)` 的全局最大值对应年份

---

### 3.3 LEAP vs KAYA 对比分析

| 维度 | LEAP 模型 | KAYA 模型 |
|---|---|---|
| 理论框架 | 自底向上，能源供需结构演变 | 自顶向下，恒等式分解分析 |
| 核心公式 | 分部门能源消费 × 排放因子 | GDP × 能源强度 × 能源碳强度 |
| 适用场景 | 能源政策分析、供需预测 | 因子贡献分解、政策效果量化 |
| 优势 | 考虑能源结构细节，预测更精细 | 逻辑清晰，便于理解各因子贡献 |
| 局限 | 需要更多分类数据，假设较多 | 过于简化，忽略技术细节 |
| 参数敏感性 | 对能效改善率和可再生能源占比最敏感 | 对产值增长率和能源强度最敏感 |
| 碳达峰判断 | 动态迭代，捕捉非线性转折 | 解析式，易于求极值 |
| 本系统使用 | 推荐用于能源结构转型情景分析 | 推荐用于政策效果量化分析 |

### 4.2 M01 — 数据管理模块

#### 4.2.1 功能描述

系统预置 2022-2024 年三年历史数据，用户可查看但本期不支持手动录入修改（数据固化）。

#### 4.2.2 数据存储规格

| 字段名 | 类型 | 说明 | 示例值（2024） |
|---|---|---|---|
| year | Integer | 年份 | 2024 |
| total_energy_consumption | Float | 综合能源消费量（万吨标准煤） | 1081.948329 |
| industrial_output_current | Float | 工业总产值现价（万元） | 10404978.32 |
| co2_total | Float | CO₂排放量合计（万吨CO₂当量） | 5273.543359 |

#### 4.2.3 派生参数（系统自动计算）

- `energy_intensity` = total_energy_consumption / industrial_output_current（能源强度）
- `carbon_intensity` = co2_total / industrial_output_current（碳强度）
- `carbon_factor` = co2_total / total_energy_consumption（单位能耗碳排放因子）
- `yoy_gdp_growth`：同比产值增长率（由历史数据计算）
- `yoy_energy_change`：同比能耗变化率
- `yoy_co2_change`：同比碳排放变化率

#### 4.3.2 情景管理操作

- **新建情景**：点击"+ 新增情景"按钮，弹出配置面板，填写参数后点击"保存"
- **编辑情景**：点击情景卡片上的编辑图标，修改参数后保存（会清除已缓存的计算结果）
- **删除情景**：点击删除图标，二次确认后删除
- **复制情景**：基于现有情景快速创建副本，便于微调参数进行对比
- **情景列表**：以卡片列表形式展示所有已保存情景，显示模型类型、关键参数值

> **约束规则**
> - 同一模块内最多同时保存 **10 个**情景
> - 情景名称不可重复
> - `start_year` 必须大于 2024（基准年）
> - `end_year` 必须大于 `start_year`，且不超过 2070
> - `gdp_growth_rate` 与 `efficiency_improvement_rate` 之差决定碳达峰是否出现（若产值增速长期远高于能效改善率且可再生能源增速极低，可能无达峰）

### 4.4 M03/M04 — 模型引擎

#### 4.4.1 触发方式

用户在情景列表页面点击顶部**"开始预测"**（或"预算"）按钮后，系统对所有已保存情景批量执行计算。单次预测计算时间不超过 3 秒。

#### 4.4.2 输出数据结构

每个情景计算完成后，生成以下逐年数据数组：

| 字段名 | 类型 | 说明 |
|---|---|---|
| year | Integer | 年份 |
| gdp | Float | 工业总产值（万元） |
| total_energy | Float | 综合能源消费量（万吨标准煤） |
| renewable_energy | Float | 可再生能源消费量（万吨标准煤） |
| non_renewable_energy | Float | 非可再生能源消费量（万吨标准煤） |
| renewable_ratio | Float | 可再生能源占比（%） |
| co2_emission | Float | 碳排放量（万吨CO₂当量） |
| is_peak | Boolean | 是否为碳达峰年份 |
| peak_label | String | 达峰标注（如"碳达峰：2038年"） |
| energy_intensity | Float | 能源强度（万吨标煤/万元） |
| carbon_intensity | Float | 碳强度（万吨CO₂/万元） |

#### 4.4.3 碳达峰判断逻辑

碳达峰年份 = 预测期内 `co2_emission` 的全局最大值所在年份，同时满足：

- 该年之后连续 **3 年**碳排放均低于该年（避免噪声误判）
- 若整个预测期内碳排放持续上升（无峰值），则标注**"预测期内未达峰"**
- 若基准年后第一年就下降，则碳达峰年份标注为**"2024年（已达峰）"**

## 5. 可视化需求详细规格

### 5.1 M05 — 预算/运算触发按钮

#### 5.1.1 UI 位置与样式

- **位置**：页面顶部操作栏右侧，固定定位于情景列表区域上方

## 附录 A：情景参数影响机制说明

以下说明帮助开发人员理解三个参数对两个模型的影响机制：

### 产值增长率（gdp_growth_rate）

> - **LEAP**：通过增加能源消费量间接增加碳排放（`E = GDP × EI`，产值增加则需更多能源）
> - **KAYA**：直接乘入公式，`C = GDP × EI × CI`，产值增速越高，碳排放越高
> - **反效果前提**：仅当能效改善率 + 可再生能源提升率的减碳效果超过产值增长带来的增碳效果，才能出现碳达峰
> - **临界条件（简化估算）**：`gdp_growth_rate < efficiency_improvement_rate + renewable_increase_rate × f₀ / (f₀ × (1-R₀))`

### 能效改善率（efficiency_improvement_rate）

> - **定义**：单位产值所消耗的能源量（能源强度 EI）每年下降的比率
> - **LEAP**：`E(t) = E(t-1) × (1 + g) × (1 - η)`，产值增长和能效提升的净效果决定能耗走势
> - **KAYA**：`EI(t) = EI₀ × (1-η)^Δt`，随时间指数下降
> - **关键阈值**：当 `η > g`（能效改善率 > 产值增速），能源消费量会绝对下降，有利于碳达峰
> - **直觉理解**：3%的能效改善率意味着在产值不变的情况下，每年节能3%，10年累计节能约26%

### 可再生能源占比提升率（renewable_increase_rate）

> - **定义**：每年可再生能源在综合能耗中的占比提升的**百分点数**（注意：是百分点，非%）
> - **举例**：初始占比5%，提升率1%/年，则第5年为10%，第10年为15%（线性增长）
> - **影响机制**：可再生能源碳排放因子 = 0，因此占比越高，单位能耗碳排放越低
> - **公式体现**：`CI(t) = f₀ × (1 - R(t))`，随可再生占比提升，碳强度线性下降
> - **量化影响**：该参数对达峰时间影响显著，从5%到30%可再生占比，碳排放减少约53%（其余条件不变）
