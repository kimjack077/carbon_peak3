## 🔧 如果前端显示问题

### 问题1：图表不显示

**可能原因**：
1. 后端未启动
2. 未运行预测
3. 浏览器缓存

**解决方法**：
```bash
# 1. 检查后端
curl http://localhost:5000/api/scenarios

# 2. 清理输出目录
rm -rf backend/output/*.png

# 3. 重新运行预测
python test_chart_generation.py

# 4. 刷新浏览器（Ctrl+Shift+R 强制刷新）
```

### 问题2：数据不对

**检查数据文件**：
```bash
# 查看当前使用的数据
head -n 3 backend/data/base_year_data.csv

# 应该显示:
# year,gdp,co2,carbon_intensity,mix_coal,...,total_energy
# 2022,754.26,4818.77,6.3887,0.8116,...,980.15
```

如果数据不对：
```bash
# 重新复制真实数据
cp backend/data/real_data_2022_2024.csv backend/data/base_year_data.csv
```

### 问题3：API错误

**检查后端日志**：
- 后端控制台会显示错误信息
- 检查 `backend/output/` 目录权限

**测试API**：

## 📝 创建自定义情景

### 通过前端界面

1. 点击"新建情景"
2. 填写参数：
   - 情景名称
   - GDP年增长率（推荐5-8%）
   - 能效提升率（推荐3-5%）
   - 化石能源降低率（推荐1-5%）
   - 可再生能源提升率（推荐1-5%）
3. 点击"保存"

### 通过API

```javascript
fetch('http://localhost:5000/api/scenarios', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    name: '我的情景',
    model_type: 'kaya',
    gdp_growth_rate: 0.06,
    efficiency_improvement_rate: 0.04,
    fossil_reduction_rate: 0.03,
    ren_share_increase: 0.03,
    population_growth_rate: 0.0
  })
})
```

## 🎯 参数建议

基于真实数据校准：

### 保守情景（缓慢减排）
- GDP增长: 7-8%
- 能效提升: 2-3%
- 化石降低: 0.5-1%
- 结果: 难以达峰或晚期达峰

### 中等情景（标准减排）
- GDP增长: 6-7%
- 能效提升: 3-4%
- 化石降低: 2-3%
- 结果: 2030-2040年达峰

### 激进情景（快速减排）
- GDP增长: 5-6%
- 能效提升: 4-5%
- 化石降低: 4-5%
- 结果: 持续下降，无峰值

### 达峰条件公式
```
CO2增长率 = GDP增长率 - 能效提升率 - 化石降低率×0.835

达峰需要: CO2增长率 ≈ 0
```
